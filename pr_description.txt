🎯 **What:** The `diff_quests` function in `src/autoscrapper/progress/update_report.py` was previously untested, leaving its core data transformation logic vulnerable to regressions. I added comprehensive unit tests to ensure it behaves correctly across all inputs.

📊 **Coverage:** The new tests cover:
- Empty before/after inputs, asserting that zero counts and empty lists are returned.
- Invalid input elements (e.g., non-dict items and quests missing an `id`), asserting they are ignored correctly.
- Additions and removals of quests, validating that items present in one list but not the other are logged correctly with relevant metadata.
- Tracked field changes (e.g., `name`, `trader`, `xp`, `sortOrder`, etc.), ensuring the `changes` dict accurately records the diff, and untracked fields do not trigger false change entries.
- No changes scenarios where matching quests with identical tracked fields result in an empty diff.
- Sorting behaviors to ensure the final changed list is sorted by name and then id as expected.

✨ **Result:** Test coverage for `diff_quests` is now at a robust level, providing a solid safety net that guarantees correct generation of changes across dataset versions and protects this critical function against future refactoring errors.
