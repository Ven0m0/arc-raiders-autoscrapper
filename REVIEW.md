# Arc Raiders AutoScrapper — PR Review Guide

> **Quick Links**: [Cross-Reference](./.kilo/CROSS_REFERENCE.md) • [Skills](./.kilo/skills/) • [Agents](./.kilo/agents/)

This is the canonical guide for AI agents performing automated code review on pull requests for Arc Raiders AutoScrapper. For development guidance, see [AGENTS.md](./AGENTS.md).

---

## Executive Summary

Automated PR reviews in this repo are driven by **specialist reviewer agents**, each scoped to a domain (OCR, TUI, Scanner, Rules, Config, etc.). Every PR change triggers one or more reviewers based on which files were modified. Review findings are reported at four severity levels:

| Severity | Meaning |
|----------|---------|
| **MUST FIX** | Blocks merge. Incorrect behavior guaranteed or data loss risk. (Reported as CRITICAL severity.) |
| **SHOULD FIX** | Strongly recommended. Likely incorrect under specific conditions. (Reported as MEDIUM severity.) |
| **CONSIDER** | Optional improvement or hygiene including style issues and code smells. Does not block merge. (Reported as LOW severity.) |
| **QUESTION** | Seeking clarification before rendering a verdict. Does not block. |

Reviewers report only concrete issues with `file:line` references. Speculative concerns are excluded unless they constitute a **MUST FIX**.

## Hotspot Reviews (Elevated Scrutiny)

The following files have critical invariants that MUST be preserved. Any change to these files requires the full standard review plus these additional checks:

### `src/autoscrapper/ocr/inventory_vision.py`

| Invariant | Description |
|-----------|-------------|
| **2× upscaling** | OCR boxes from a 2×-upscaled image must be halved before reuse in original-space operations |
| **Main-thread OCR init** | `initialize_ocr()` must run on the main thread before any scan threads start |
| **No direct BGR→Tesseract** | Raw BGR frames must pass through `preprocess_for_ocr()` (Otsu binarization + morphology) before OCR |
| **Cache reset at scan start** | `reset_ocr_caches()` must be called at entry to `scan_pages()` |

### `src/autoscrapper/ocr/tesseract.py`

| Invariant | Description |
|-----------|-------------|
| **Four Tesseract API locks** | `_api_lock`, `_api_line_lock`, `_api_single_word_lock`, `_api_sparse_lock` — each dedicated to one PSM mode, never shared across modes |

### `src/autoscrapper/scanner/scan_loop.py`

| Invariant | Description |
|-----------|-------------|
| **Page mode isolation** | `_detect_ui_mode()` resets per-page via `_ScanRunner._detected_ui_mode` — state must not leak across pages |
| **Dry-run gate** | `--dry-run` must exercise the full decision pipeline (OCR → fuzzy match → rule lookup → action resolve) but skip all click/input calls only |
| **Cache reset at entry** | `reset_ocr_caches()` must not be removed or moved past the first OCR call |

### `src/autoscrapper/core/item_actions.py`

| Invariant | Description |
|-----------|-------------|
| **Custom rule precedence** | User-defined custom rules always override default rules |
| **Fuzzy threshold coupling** | Threshold used in OCR name matching must equal the threshold used in rule lookup |
| **Defined fallback** | No-rule-matched defaults to `keep`, not `None` |

### `src/autoscrapper/items/rules_store.py`

| Invariant | Description |
|-----------|-------------|
| **Generated file guard** | `items_rules.default.json` is generated by `scripts/update_snapshot_and_defaults.py` — direct edits are rejected |
| **Error-tolerant loading** | Missing custom rule file produces empty set; missing default file is fatal |

### `src/autoscrapper/config.py`

| Invariant | Description |
|-----------|-------------|
| **Version bump on field change** | Any new, removed, or renamed dataclass field requires `CONFIG_VERSION` bump + migration |
| **JSON round-trip** | All fields must serialize/deserialize through `json.dumps`/`json.loads` natively |

### `src/autoscrapper/progress/data_update.py`

| Invariant | Description |
|-----------|-------------|
| **Generated file guard** | `src/autoscrapper/progress/data/*` and `items_rules.default.json` written only by `update_data_snapshot()` and `scripts/update_snapshot_and_defaults.py` |
| **Dual-source fallback** | Pipeline raises only if both MetaForge and RaidTheory sources fail |
| **Normalization stability** | Changing `_normalize_entry_name` alters deduplication identity — flag any impact |

## Validation Requirements by Change Type

A PR that claims a behavior is validated must provide the evidence below. Reviewers block unverifiable claims.

| Change type | Required evidence |
|-------------|------------------|
| Python source (general) | `uv run ruff check src/ tests/ scripts/` · `uv run basedpyright src/` · `uv run pytest` — all passing |
| Workflow files (`.github/workflows/*.yml`) | `uv run prek run --files .github/workflows/<name>.yml` — all hooks passing |
| OCR / scanner / input changes | Live dry-run scan: `uv run autoscrapper scan --dry-run` against a live game window. **CI cannot substitute for this.** |
| Config field changes | CONFIG_VERSION bumped + migration written + round-trip test included |
| Fuzzy threshold changes | `/ocr-corpus-replay` completed; regression report attached |
| Generated data changes | Script used with `--dry-run`; diff reviewed against canonical upstream source |
| New action / click behavior | `security-reviewer` sign-off required |

## Review Severity Guidelines

### MUST FIX

- **Breaking a documented invariant** listed in the Hotspot Reviews section
- **Hand-edit of generated files** (`progress/data/*`, `items_rules.default.json`)
- **Missing config migration** when a new field is added to a persisted dataclass
- **Threading violation** in TUI (`call_from_thread` missing on cross-thread widget mutation)
- **Capture-space / screen-space coordinate mixing** (image coords used as click targets)
- **2× upscale bbox reuse without halving** (coordinate regression in OCR output)
- **Missing dry-run gate** on input/click calls
- **Custom rule override by default rule** (precedence inversion)
- **Divergent fuzzy thresholds** between OCR matching and rule lookup
- **Silent empty rule set** from error-swallowing load path

### SHOULD FIX

- **Hardcoded pixel constants** that should be normalized to image dimensions (use `img_w`/`img_h` fractions)
- **np.ndarray shape assumption** (`shape[2]` on potentially grayscale image without guard)
- **Missing `await` on async def** in Textual event handlers (silently drops coroutine)
- **Unguarded API dict access** (direct `.get()` on response without `(response or {})` wrapping)
- **Worker not cancelled on screen unmount** (risk of `NoActiveAppError`)
- **Window handle used across scan iterations** without re-validation
- **Pagination missing `hasNextPage` fallback** when `totalPages` absent
- **Error swallowed without propagation** to fallback error state or logging

### CONSIDER

- Repeated `cv2.cvtColor` on the same frame within a single scan tick
- Unnecessary `np.copy()` in hot-path functions
- Any `Any` escape in type annotations within performance-critical paths
- Missing docstring on public API methods
- Hardcoded numeric magic numbers without named constants

## Reviewer Reference Cards

### ocr-reviewer

**Scope**: `src/autoscrapper/ocr/`, `src/autoscrapper/scanner/`

Focus: Coordinate space integrity, Tesseract configuration ordering, cache invalidation, empty-slot detection logic.

Key checks:
1. 2× upscale bbox halved before original-space reuse
2. Otsu binarization before morphological operations
3. New module-level cache has `reset_ocr_caches()` entry
4. `slot_bgr.size == 0` guards in `_find_first_empty_slot` do not modify `prev_empty`

---

### tui-reviewer

**Scope**: `src/autoscrapper/tui/`

Focus: Textual framework correctness, threading discipline, reactive state mutations, screen lifecycle.

Key checks:
1. All widget mutations from background threads wrapped in `app.call_from_thread()`
2. `@work` workers cancelled on screen unmount
3. Reactive collections mutated through assignment path, not `.append()`/`.extend()` directly
4. All `async def` calls in event handlers are `await`ed
5. Screen push/pop balanced (every push has a dismiss path)
6. Message handler names match widget message classes exactly (case-sensitive, underscores)

---

### scan-validator

**Scope**: `src/autoscrapper/scanner/`, `src/autoscrapper/interaction/`

Focus: Timing, page detection, action dispatch, window targeting.

Key checks:
1. `time.sleep()` delay changes documented with justification
2. `_detected_ui_mode` reset per page (no cross-page state leak)
3. `_dispatch_action()` maps keep/sell/recycle correctly; dry-run skips input only, not decision logic
4. Cached window handles re-validated before use across iterations
5. Window-relative rects not used directly as screen-space click targets
6. `reset_ocr_caches()` at `scan_pages()` entry
7. Dry-run exercises full pipeline but gates clicks only

---

### interaction-reviewer

**Scope**: `src/autoscrapper/interaction/`

Focus: Platform guards, grid detection, coordinate transforms, dry-run safety.

Key checks:
1. Platform-specific code properly guarded (Windows vs Linux)
2. Click coordinates validated within bounds before execution
3. `--dry-run` respected to prevent accidental clicks
4. Input queue flushed after actions
5. Grid cell rects in window-relative coords translated before screen-space use

---

### rules-reviewer

**Scope**: `src/autoscrapper/core/item_actions.py`, `src/autoscrapper/items/rules_store.py`

Focus: Rule precedence, action mapping, fuzzy matching consistency, fallback correctness.

Key checks:
1. Custom rules override default rules (no silent inversion)
2. Action enums (`keep`, `sell`, `recycle`) map to distinct codes
3. No-rule-matched resolves to explicit `keep`, not `None`
4. Error-tolerant loading does not silently discard custom rules on file-not-found
5. Generated `items_rules.default.json` not directly edited
6. Progress-aware overrides do not suppress explicit user sell rules
7. Same threshold constant used in OCR fuzzy match and rule lookup

---

### config-reviewer

**Scope**: `src/autoscrapper/config.py`

Focus: Versioning, migrations, serialization integrity, platform path handling.

Key checks:
1. New field added → CONFIG_VERSION bumped
2. Migration or default-fill path exists for configs lacking the new field
3. All fields round-trip through `json.dumps`/`json.loads` without custom encoder
4. Platform path logic handles both Windows (`~/.AutoScrapper/`) and Linux (`~/.autoscrapper/`)

---

### data-pipeline-reviewer

**Scope**: `src/autoscrapper/progress/data_update.py`

Focus: Field mapping, merge deduplication, error propagation, generated-file guard, pagination.

Key checks:
1. Hardcoded API key strings (`"itemId"`, `"item_type"`) not silently broken by upstream rename
2. Normalization changes do not alter merge identity (previously merged items re-appearing or dropped)
3. New `try/except` blocks propagate to fallback error state or log
4. Pagination handles `hasNextPage` fallback when `totalPages` absent
5. Generated files written only by `update_data_snapshot()` or updater script
6. Component map key fields (`item_id`, `component_id`, `quantity`) not silently broken upstream

---

### progress-reviewer

**Scope**: `src/autoscrapper/progress/` (excluding data_update.py)

Focus: Generated-file bypass, quest inference, crafted-item overrides, data loader guards.

Key checks:
1. `items_rules.default.json` written only by the generator script
2. Newly added quests reachable by inference logic, not silently excluded
3. Crafting/hideout overrides do not suppress explicit user sell rules
4. JSON key accesses guarded with `.get()` or explicit `KeyError` handling
5. Rules generator output conforms to schema expected by `rules_store.py`

---

### api-reviewer

**Scope**: `src/autoscrapper/api/`

Focus: Slot mapping, None-guard coverage, coordinate plumbing, HTTP error handling.

Key checks:
1. Fixed 4×5 grid (20 items/page) expressed as named constant — derived calculations consistent
2. All `.get()` on API responses use `(response or {}).get(...)` pattern
3. Slot fallback `idx` is iteration index (0-based), not page-relative
4. `Cell(page, row, col)` all 0-based and consistent with `scanner/types.py`
5. HTTP methods handle non-200 responses without crashing the scan

---

### performance-reviewer

**Scope**: `src/autoscrapper/ocr/inventory_vision.py`, `src/autoscrapper/scanner/scan_loop.py`, OCR preprocessing paths.

Focus: Redundant copies, lock contention, per-cell overhead, type annotation gaps.

Key checks:
1. Unnecessary `.copy()` calls on NumPy arrays in hot path
2. Repeated `cv2.cvtColor` on the same frame within a scan tick
3. Tesseract locks not held during preprocessing — preprocessing completes before lock acquisition
4. Narrowest PSM instance used per caller (not over-generalizing to SINGLE_BLOCK)
5. No nested lock acquisition (deadlock risk)
6. Per-cell OCR calls that could be batched or skipped with empty-cell pre-check
7. Frame capture (`mss`) outside per-cell loop
8. `Any` escapes or missing return types in hot-path functions

---

### security-reviewer

**Scope**: `src/autoscrapper/interaction/`, `src/autoscrapper/scanner/`, `src/autoscrapper/core/item_actions.py`, new automation patterns.

Focus: Input validation, capability checks, action safety, window targeting integrity.

Key checks:
1. Click coordinates validated within bounds
2. Cell/item validity checked before click
3. Window focus verified or gracefully skipped
4. No injection-like patterns in keystroke conversion
5. Platform detection (Windows vs Linux) guards input method usage
6. Required imports available before use — graceful failure in dry-run
7. User opt-in required before automated clicks
8. Safe action precedence maintained (KEEP > SELL > RECYCLE)
9. No accidental rule inversions
10. Correct window detection (game vs other apps); focus restored after interaction
11. Timeout/fallback prevents hangs if window lost mid-action

Approve checklist:
- [ ] All click coordinates validated within bounds
- [ ] Input method availability checked (platform-specific)
- [ ] Rule resolution precedence unchanged
- [ ] Dry-run mode blocks actual clicks
- [ ] Window focus verified or gracefully skipped
- [ ] No hardcoded window names
- [ ] Input timeout/retry logic present

---

### test-generator

**Trigger**: After a PR passes all reviewer sign-offs. Generates pytest cases for new or modified Python source.

Placement: `tests/autoscrapper/<module>/test_<file>.py`

Coverage requirements:
- Happy path with representative inputs
- At least one edge case per public function (empty, boundary, failure)
- OCR functions: mock `tesserocr`; test coordinate conversion directly
- Scanner functions: mock `interaction`; test state transitions and rule application
- API functions: mock HTTP with `pytest-httpx`; test retry and error paths
- `item_actions.py`: test fuzzy thresholds with near-miss item names

## Review Workflow Chains

### Standard PR Review
```
Identify changed files → dispatch applicable reviewer(s) → aggregate findings by severity →
MUST FIX blocking → SHOULD FIX recommendations → CONSIDER suggestions → pass/block decision
```

### Failed Review — Fix Round-Trip
```
Reviewer reports findings → author addresses → re-request review → same reviewer re-invokes →
iterate until all MUST FIX resolved
```

### Multi-Domain PR (Complex)
```
Change spans OCR + rules + config → dispatch ocr-reviewer + rules-reviewer + config-reviewer →
consolidate findings → single unified review output per severity level
```

### Hotspot-Changed PR
```
File in Hotspot list → dispatch primary reviewer + performance-reviewer (+ security-reviewer if interaction) →
elevated scrutiny on documented invariants
```

## Comment Style Conventions

Use one of four labels in review comments:

| Label | When to use |
|-------|------------|
| `MUST FIX` | Blocks merge. Severity: HIGH. Includes file:line reference and concrete fix. |
| `SHOULD FIX` | Recommended. Severity: MEDIUM. Explain condition under which bug manifests. |
| `CONSIDER` | Optional improvement. Severity: LOW. Suggest alternative without blocking. |
| `QUESTION` | Seeking clarification before rendering a verdict. Does not block. |

Format: `file:line — [MUST FIX|SHOULD FIX|CONSIDER|QUESTION]: description of issue`

Example:
```
src/autoscrapper/ocr/inventory_vision.py:142 — MUST FIX: bbox from OCR over 2× image used directly as click target without halving. Will cause ~2× offset misclicks in original-space operations.
```

## Blocking Conditions (PR Cannot Merge)

A PR is blocked if ANY of the following are true:

1. A **MUST FIX** finding is unresolved
2. A hotspot file changed and the relevant invariant was violated
3. A config field was changed without a version bump and migration
4. A generated file was directly edited (not via the updater script)
5. An OCR/scanner/input change claims live-window validation but no evidence is provided
6. A fuzzy threshold change ships without corpus replay results
7. A workflow file change fails `uv run prek run --files .github/workflows/<name>.yml`
8. `ruff check`, `basedpyright`, or `pytest` do not pass

## Before Merging

Verify these pre-merge gates:

| Gate | Command |
|------|---------|
| Lint | `uv run ruff check src/ tests/ scripts/` |
| Types | `uv run basedpyright src/` |
| Tests | `uv run pytest` |
| Hooks | `uv run prek run --all-files` |
| Fork sync | `/upstream-sync` (syncs fork from upstream before merge) |
| Branch up-to-date | Rebase or merge `main` into branch |
